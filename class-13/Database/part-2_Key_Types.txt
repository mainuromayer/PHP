Types of Key -
	-> Primary Key
	-> Composite Primary Key
	-> Foreign Key


-----------------------------------------------------------------------------


Primary Key :
============
■ যে Field-র সাহায্যে প্রতিটি Recordকে পৃথকভাবে সনাক্ত করা যায়, তাকে Primary Key বলে।

Example :

  𝗣𝗿𝗶𝗺𝗮𝗿𝘆
   𝗞𝗲𝘆
+--------+--------+--------+-------+
| 𝗥𝗼𝗹𝗹    | Name	  | Gender |  Age  |
+--------+--------+--------+-------+
| 𝟭𝟬𝟭    | Rahim  | Male   |  18   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟮    | Hasina | Female |  17   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟯    | Rahim  | Male   |  18   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟰    | Hasina | Female |  17   |
+--------+--------+--------+-------+

Note : Roll Primary Key, Because This is Unique



-----------------------------------------------------------------------------



Composite Primary Key :
======================
■ অনেক সময় একটি Field-র  সাহায্যে প্রতিটি রেকর্ডকে পৃথকভাবে শনাক্ত করা যায় না, 
    তখন দুই বা ততোধিক Key Field-র সাহায্যে Composite Key তৈরি করে প্রতিটি Record কে পৃথকভাবে সনাক্ত করা যায়।

Example :

	     𝗖𝗼𝗺𝗽𝗼𝘀𝗶𝘁𝗲 𝗞𝗲𝘆
		  ↑
(𝗥𝗼𝗹𝗹-𝗙𝗲𝗶𝗹𝗱) --------------- (𝗔𝗴𝗲-𝗙𝗲𝗶𝗹𝗱)

+--------+--------+--------+-------+
| 𝗥𝗼𝗹𝗹    | Name	  | Gender |  𝗔𝗴𝗲  |
+--------+--------+--------+-------+
| 𝟭𝟬𝟭    | Rahim  | Male   |  𝟭𝟴   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟮    | Hasina | Female |  𝟭𝟴   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟭    | Rahim  | Male   |  𝟭𝟳   |
+--------+--------+--------+-------+
| 𝟭𝟬𝟮    | Hasina | Female |  𝟭𝟳   |
+--------+--------+--------+-------+

Note : Roll Not Unique. So Roll with another feild make Composite Key



-----------------------------------------------------------------------------



Foreign Key :
=============
■ একটি টেবিলের Primary Key যদি অন্য টেবিলে সাধারণ Key হিসেবে ব্যবহার করা হয়, 
তখন সেই সাধারণ Key কে, Foreign Key বলে।

Example :
		A						B

  𝗣𝗿𝗶𝗺𝗮𝗿𝘆						  𝗣𝗿𝗶𝗺𝗮𝗿𝘆	       Foreign
   𝗞𝗲𝘆						   𝗞𝗲𝘆	        Key
+--------+---------+--------+-------+		+----------+------------+---------+
| 𝗥𝗼𝗹𝗹    | Name	   | Gender |  Age  |		|  𝗥𝗲𝗴𝗡𝘂𝗺  |	Roll	|   GPA	  |
+--------+---------+--------+-------+		+----------+------------+---------+
| 𝟭𝟬𝟭    | Rahim   | Male   |  18   |		|  𝟭𝟬𝟭𝟭𝟭𝟮  |	101	|   3.25  |
+--------+---------+--------+-------+		+----------+------------+---------+
| 𝟭𝟬𝟮    | Farjana | Female |  17   |		|  𝟭𝟬𝟮𝟭𝟭𝟯  |	102	|   3.44  |
+--------+---------+--------+-------+		+----------+------------+---------+
| 𝟭𝟬𝟯    | Mahfuz  | Male   |  18   |		|  𝟭𝟬𝟯𝟭𝟭𝟰  |	101	|   4.00  |
+--------+---------+--------+-------+		+----------+------------+---------+
| 𝟭𝟬𝟰    | Farjana | Female |  17   |		|  𝟭𝟬𝟰𝟭𝟭𝟱  |	102	|   5.00  |
+--------+---------+--------+-------+		+----------+------------+---------+

Note : Section A : Roll is Primary key.
and Roll is use another section or Section B in Generaly.
So This is Foreign Key case Section B.

Section A : Primary Key  => Roll
Section B : Primary Key  => RegNum
Section A Primary Key use Section B Foreign Key => That means two table is Relational
